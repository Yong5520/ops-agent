# OpsAgent - 项目任务执行跟踪

> Phase 1 MVP 任务拆解，按模块分组

## 状态说明

| 状态 | 含义 |
|------|------|
| `pending` | 待开始 |
| `in_progress` | 进行中 |
| `done` | 已完成 |
| `blocked` | 被其他任务阻塞 |

## 复杂度说明

| 标记 | 含义 |
|------|------|
| `S` | 简单（< 2h） |
| `M` | 中等（2-8h） |
| `L` | 复杂（1-3 天） |
| `XL` | 非常复杂（3-5 天） |

## 执行建议

1. **并行策略**：M1 完成后，M2/M3/M4 可并行开发（互不依赖）
2. **关键路径**：M5-06（Agent 主循环）是整个项目的关键瓶颈，建议尽早开始
3. **先通后精**：先让最简单的对话流程跑通（M1 → M2 → M3 → M5 → M7 最小集），再补全安全、审计等模块
4. **可复用优先**：M3（SSH 层）和 M4（安全层）有大量代码可从 ssh-mcp-multi 提取，应优先完成
